# Security Policy

The security of Chain of Lands is a priority.

Although the project is currently in an experimental and early-stage phase,
responsible disclosure and proactive security awareness are encouraged.

---

## 1. Scope

This security policy applies to:

- Public repositories related to Chain of Lands
- The interactive map frontend
- The Eru AI narrative engine
- Documentation and tooling associated with the project

Smart contracts and blockchain integrations,
when introduced, will be covered under an expanded policy.

---

## 2. Reporting a Vulnerability

If you discover a security vulnerability, please follow these steps:

- **Do not** open a public GitHub Issue
- **Do not** disclose the issue publicly
- Provide a clear and detailed description of the vulnerability

Please report vulnerabilities privately by:
- Contacting the repository maintainers via GitHub
- Including reproduction steps if possible

Responsible disclosure is appreciated and respected.

---

## 3. What to Include in a Report

A good vulnerability report should include:

- A clear description of the issue
- Affected files or components
- Steps to reproduce the vulnerability
- Potential impact assessment

Incomplete reports may delay response time.

---

## 4. Response Process

Upon receiving a valid report:

- The issue will be reviewed and validated
- Mitigation or remediation steps will be planned
- Fixes will be applied in a controlled manner
- Public disclosure may occur after resolution, if appropriate

Response times may vary depending on severity and project stage.

---

## 5. Supported Versions

At this stage, only the **latest version** of the project
is actively supported for security updates.

Older versions may not receive patches.

---

## 6. Security Philosophy

Chain of Lands approaches security with the following principles:

- Minimize unnecessary complexity
- Isolate critical components
- Favor clarity over obscurity
- Design for graceful failure

Security is treated as an ongoing process,
not a one-time checklist.

---

## Closing Statement

Security is essential to trust,
and trust is essential to persistence.

Chain of Lands values responsible collaboration
in keeping the world stable and resilient.
